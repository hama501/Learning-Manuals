# GPIO

## GPIOとはなんぞや

GPIOとは、電流を流すか流さないかのいわゆるON・OFFで制御をする方法です。主にマブチモーターの回転方向を変えたり、LEDを点滅させるときに使います。あと電源基盤。
GPIOはめっちゃ簡単なのでサクッと行きましょう！

まずはPIN設定です。GPIOのPIN設定はシンプルで、どこのPINから電流を流すのか、受け取るのかを決めるだけです。今受け取ると聞いて疑問に思ったかもしれないですが、GPIOにはinputと**output**があります。outputは電流を流す・流さないという動作を行うもので、inputは基盤に回路班がつけた機能によって電流が流されているか流されていないかを読むものです。基本outputのほうを使うのですが、電源基盤ではinputも多少使うので一応書いておきました。

## コーディング

PIN設定ができたらいよいよコーディングです。基盤設定を保存して、コードを書く画面に移りましょう。

GPIOを使うときは、PWMなどと違い、スタートさせる必要がありません。要するに、本当にONとOFFを切り替えるだけです。一応書いておきますが、ONが電流を流す、OFFが止めるです。

1.GPIOのON(AorB, PINの番号)
```c
HAL_GPIO_WritePin(GPIOAorB, GPIO_PIN_?, GPIO_PIN_SET);
```
2.GPIOのOFF(AorB, PINの番号)
```c
HAL_GPIO_WritePin(GPIOAorB, GPIO_PIN_?, GPIO_PIN_RESET);
```

これがGPIOで使う関数です。まずはコピペせず自分で書いてみよう！

…え？書く場所がわからないって？これらを書く場所は用途によって変わってきます。一度流せばOKならONの関数を独自の関数として定義し、それをBEGIN 2で実行します。なぜ2かはわかりますよね？

また、ONとOFFを一定の間隔で切り替えたい場合は、while文(USER CODE BEGIN WHILE)の中にONとOFFを切り替えるコードを書きましょう。その時に必要になるDelay関数は、関数一覧の中にのせています。もちろん自分で調べてもいいですよ！
